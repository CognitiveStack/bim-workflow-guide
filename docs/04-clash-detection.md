# 04 — Clash Detection

> **Status: coming soon (Phase 2).**

One step inside coordination: automatically finding where elements **physically conflict** or break clearance rules.

This document will follow the standard topic template:
- **What it means** — hard clashes, clearance clashes, workflow clashes.
- **Why it matters** — catching conflicts in the model, not on site.
- **Where it fits** — after coordination review, feeding issue management.
- **Typical tools** — Navisworks, Forma Model Coordination.
- **Simple example** — detecting a pipe through a doorway in the fire station.

> Remember: clash detection is **one activity** inside the wider workflow, not the whole thing.
